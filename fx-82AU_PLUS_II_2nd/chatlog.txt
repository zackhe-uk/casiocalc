303### some idea about gadget finding 

* Example gadget '√∑-00' correspond to '4f 2d 30 30' which point to '0:2d4fh' which prints 3 lines of string to the screen.
* Now, to actually verify that '0:2d4fh +/- 1' is the start of the multi line print function we'll decrease the address by one to get '0:2d4eh' ( √ó - 0 0 ).
* If it still prints 3 lines, we continue decreasing the address by 1 until we reach an address that will no longer execute the print function.
* If it doesn't print 3 lines or it doesn't print at all, it means '0:2d4f +\- 1 is the start of the function.
* When we reach the address that doesn't print lines , we increment the address by 1 so that we reach the last address that worked, and that is the start of the function.

###Example of  start of a function 
`
printnlines:

0:2d4eh.            ;
0:2d4fh.            ;This is where you guys usually start executing the print from
0:2d50h.           ; another command 
0:2d52h.           ; another command 
`
i use 0:2d??h because we don't know where the function starts, if you keep decreasing the address until it doesn't print, the address after the address that doesn't print is the start address of the function.
